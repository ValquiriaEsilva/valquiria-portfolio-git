#REFLEXÃO

## O que foi difícil

No começo, a parte mais difícil foi entender como usar os comandos básicos do git, principalmente o fluxo de add, commit e push. Também tive dificuldade com o terminal, porque qual erro de digitação já fazia o comando não funcionar.
Outra parte que achei difícil foi trabalhar com branches. No ínicio eu me confundi muito com os nomes das branches e não entendia bem em qual eu estava. Isso ficou claro quando usei o comando git branch para verificar.
A parte mais complicada foi o conflito. Eu não entendia bem por que o conflito acontecia´, fiquei meio perdida com os símbolos que apareciam. Mas depois que entendi que eram duas versões do mesmo trecho, consegui resolver.



## O que ficou claro

Depois de praticar, ficou bem claro para mim como funciona o fluxo básico do git: fazer alteração, usar git add, commit e finalizar com git push.
Tambem entendi melhor o conceito de branch, como uma forma de trabalhar em paralelo sem afetar a versão principal do projeto.
Além da importância das mensagens de commit. Percebi que escrever commits descritivos ajuda muito a entender o que foi feito.



## O que ainda é confuso

Ainda tenho algumas dúvidas sobre quando usar o merge ou outras formas de integrar mudanças, como rebase.
Também não entendo totalmente como o git gerencia o histórico internamente, mas sei usar na prática.
Além disso, ainda preciso praticar mais para me sentir mais segura usando git sem ajuda, principalmente em situações com erro.
